What is this

This is a load manager simulator/analyzer of a very specific and neash area.

In Spain, if you have renewable energy porduction, your energy bill goes as follows: At the end of every hour your energy balance is calculated (energyConsumed - energyProduced).

  1. If you have consumed more energy that produced, you must pay that difference. Which can get very expensive depending on the hour.
  2. If instead you have produced more that consumed, the company has to pay. But the price they pay is around half of the lowest hour price, which is not very good.

Thanks to this hour slot system you can, for example, consume 1 kWh of energy in the first 10 min and then produce 1kWh during 20 min, thus having a net zero consumption and not paying anything.

This project aims to create, analize and compare diferent algorithms which manag loads adequately so to achieve: the màximum energy bill price reduction (try to get to 0), maximise the use of the energy produced, and minimize the number of load commutations.

This simulator has been implemented to test and analize the algorithms. With it, the best settings can be found before implementing the system in real life.

Load File

The input files used for simulation or calculating results are .csv. To load you click the 'Load File' button.

For simulation

If simulating, the .csv must at least have the columns:

  • timestamp → string of datetime (ex: 2022-02-01 15:57:50)
  • powerP → float (stands for powerProduced)

For real data results

If loading real data for calculating the results, the .csv must have the previous columns plus the columns that would be filled by the simulation:

  • powerC → float (stands for powerConsumed) (= sum of all loads power)
  • powerA → float (stands for powerAvailable) (= powerP - powerC)
  • powerL1 → float (stands for powerLoad1)
  • powerL2 → float (stands for powerLoad2)
  • on_offL1 → [0, 1, -1] (no change, turned on, turned off)
  • on_offL2 → [0, 1, -1] (no change, turned on, turned off)


Some optional columns, which will speed the calculations are:


  • powerLB → float (stands for powerLoadBase)
  • energyP → float (stands for energyProduced)
  • energyB → float (stands for energyBalance)


When doing the calculations some columns are added or changed:

  • powerA → float (stands for powerAvailable) (= positive(powerP - powerC))
  • powerG → float (stands for powerAvailable) (= -negative(powerP - powerC))
  • energyAB → float (stands for energyAvailableBalance) (= positive values of energyB)
  • energyGD → float (stands for energyGridDebt) (= negative values of energyB with sign changed)

Settings

All the settings are located at the left side column section of the program.

Date & Time

Indicates the day and time from where the simulation starts and ends. This values range are limited by the dataset give.

You can use the scroll wheel to increment/decrement the value. You can also select the dates ranges using the plot in Date Range:

  • Select a date line to move it
  • Pres ESC to cancel the move

Algorithm

You can select what algorithm to simulate and its associated settings. They can control up two two loads.

Hysteresis

As the names says, loads are controlled using hysteresis. It allows for up two loads to be managed. Each top and bottom threshold corresponds to the load with the same number. If the load is 0, the thresholds get disabled.

Min On Use

If one load is controlled it's turned on when the time it would take to consume the energy accumulated is equal or greater than the time indicated, and will stay on until that time has passed. It's equivalent to having a single threshold (which is also calculated for reference). While the load is on, it keeps doing the check and refreshing the timer.

The same applys to two loads. First it check the time it would take to consume if turning on both loads. If they can't be both turned on, but any of both by themselfs can, it turns on one load depending on the selected mode:

  • 2n on if 1st on → load 2 only turns on if the 1st load is on. This avoids the problem of both loads being able to turn one if the other doesn't.
  • By order → load 1 has preference (this is not the same as the previous, since load 2 can be turned on if load 1 isn't)
  • Fastest → turn on the load that consums the energy fastest

Finally, if both prebious conditions aren't true but one load can be turned on, it's turned on.

Time to consume

If one load is controlled it's turned on when the time it would take to consume the energy accumulated is equal or greater than the time left to end the hour. You can also enable the predict final energy modes which predicts how much energy that hour slot will probably yield (energy1h). This value is calculated every sample. There are two modes:

  • Average Power → avgP = energyB / (3600 - sec_remaining) -> energy1h = avg*3600
  • Project Current Power → energy1h = energyB + timeLeft * powerP / 3600

You can also configure:

  • End at → indicates how much energyB should be left when the hour ends (recommended at 0 or more, so you don't pay/have a negative hour energy balance)
  • On Min → while the energyB is less than this value, the load cannot turn on.
  • Time Factor → this values gest multiplied to the time left to end the hour. This means, that if set to less than 1, it will turn on earlier since it "will think that is has less time". This is useful, for example, if the prediction is off and a bunch more energy is produced, if this time shift wasn't applied that energy would be lost.

Calculating energyB

Because of how energy is calculated, at each hour the load doesn't always turn off.

The formula is energyB = ( powerP - powerC ) * Ts / 3600 + energyB, which means we are supposing/"predicting" that after Ts will have produced energyP = ( powerP - powerC ) * Ts / 3600 Wh.

If we instead did energyP = ( powerP(-1) - powerC(-1) ) * Ts / 3600 and at t=0 energy = 0 we would not have this prediction. We might think this can be problematic, but in fact is the opposite, since it vastly improves the algorithms by avoiding a lot of possible commutations because of the fact that we don’t start at 0 Wh.

Loads

It's where you indicate the loads values for simulating.

  • Load X Depending on the number of loads that the algorithm supports, the fields get disabled.
  • Base Load represent the constant power draw of a building. It can be 0 if needed to not factor it.
  • Use base load If checked and the data you loaded has the powerLB, the simulation will use those values. If not checked or no values exist, it will use the number indicated.

Generation Factor

When simulating, power generated is multiplied by this value. This can be usefull for simulating diferent situations: seasons, cloudy, having less or more solar panels, ...

Input Tabs

⌚ Data Range

Used for seeing the input data (power generated) from the .csv. You can also easily change the date by interacting with the plot:

  • Click on a date line to select it.
  • Click again to set it.
  • Pres 'ESC' to set the value to the original position.

💰 Energy Price

Used for setting and visualizing the price of the energy at every hour. There are two ways of setting the values

  • Using the plot
  • Click any point or line to select it.
  • Click again to set it.
  • Pres 'ESC' to set the value to the original position.
  • Using text inputs
  • Double click or select+F2 to edit the value on the table
  • Set the sell price using the number input

Output Tabs

📊📅 Results

After simulating or calculating results, this tab contains the important results. It also has two plot settings that are very useful:

  • Values → displays a label next to each bar/area value
  • System Max Consuption → displays the max consumption that the system could achieve
  • Subdivide → each bar is subdivided in its relevant data parts

📉 Simulation

After simulating or calculating results, is filled with a plot of the simulated data. It also has three plot settings:

  • Power Consumed by Loads → shows the loads plots as a stacked area, this helps see the components of Power Consumed.
  • Power Produced → shows all the energy produced which.
  • Show threshold → displays horizontal lines corresponding to each threshold (works in "hysteresis" and in "min on use").

📊 Hourly Results

After simulating or calculating results, contains the hourly breakdown of the energy, commutations, efficiency, ... Both plot settings in summary are also available here.

📈 Optimize

Used to simulate the dataset multiple times, each time incrementing by the step the indicated setting value. After that, a plot is displayed where you can contrast different relevant data (checkboxes). This is very useful for finding the best algorithm configuration for your case.

Interactive Plots

The plots are made matplotlib, this means that they have all the matplotlib funcionalities. But apart from that, some extra options have been added.

  • Cross Hair Cursor → Toggled by selecting the toolbar buttons + it displays a cross hair which snaps to the closest point of the data line. To change the line it snaps, you simply have to select it.
  • Line Style → A input that accepts any matplotlib line style and marker to apply to the displayed lines.
  • Hide lines → Any line in a plot can be hidden by pressing the lines in the legend. Hidden lines still show in the legend but more transparent.
